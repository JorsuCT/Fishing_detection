# Local Engine: Benchmarking & Analytics

**Tags / Tech Stack:** `Python` | `Pandas` | `Seaborn & Matplotlib` | `Paramiko (SSH)` | `scikit-learn` | `Edge Orchestration`

This directory contains the client-side logic of the project. It is responsible for traversing the dataset, sending inference requests to the backend, evaluating the semantic responses, and generating visual analytics to compare model performance.

## Key Components

### 1. Benchmark Execution (`benchmark.py` & `benchmark_vqa.py`)
These are the main scripts to run evaluations across the image dataset. 
* They iterate recursively through the dataset, dispatch images to the API via `server_controler.py`, and compute the accuracy of the model's output against the folder-based ground truth.
* `benchmark_vqa.py` is specifically optimized for direct Yes/No Visual Question Answering (VQA) prompts.
* **Usage Example:** ```bash
  python benchmark_vqa.py "Qwen/Qwen3-VL-2B-Instruct" "DeepLearning2"
  ```

### 2. Edge & Server Orchestration
* **`server_controler.py`**: Handles HTTP REST requests to the FastAPI backend. It converts Pillow images to byte streams, manages the network connection, and processes the JSON responses.
* **`jetson_controler.py`**: A specialized orchestrator for edge deployment on NVIDIA Jetson boards. It controls the local Docker daemon via SSH (Paramiko) to start and safely shut down containerized models on demand. This ensures a pristine VRAM state between evaluations and prevents memory leaks.

### 3. NLP Evaluation (`evaluation.py` & `fishing_evaluation.py`)
Scripts used to interpret complex, open-ended descriptions generated by the models. Instead of simple keyword matching, they utilize NLP semantic matching (e.g., Sentence Transformers like MiniLM) to determine if a generic image caption implies fishing activities, providing a much more robust evaluation metric.

### 4. Data Visualization (`plot_*.py` scripts)
A suite of tools designed to convert the generated CSV metrics into publication-ready graphs for the thesis:
* **`plot_metrics.py`**: Generates Bar Charts comparing basic F1-Scores and Execution Times.
* **`plot_experimentos.py`**: Evaluates "Prompt Sensitivity" by grouping results based on the Prompt Strategy (e.g., Description, Yes/No, Multiple Choice) parsing the CSV suffixes.
* **`plot_seleccion_modelo.py`**: Generates Pareto-style Scatter Plots (Time vs. F1-Score) to objectively calculate an Efficiency Score and select the optimal model for edge deployment.

## Folder Structure Expected
To run the benchmarks seamlessly, ensure your image dataset is placed at the root level relative to where you execute the scripts:

```text
dataset/
├── fishing_rod/
├── fishing_net/
└── no_fishing/
```

## Generated Output
During benchmark execution, a `metrics/` folder is automatically created. It stores detailed CSV files for every individual prediction to allow debugging and auditing, alongside a centralized `model_performance_summary.csv` that accumulates the global results utilized by the plotting scripts.
